**4. Create `lib/features/notifications/presentation/screens/notification_settings_screen.dart`:**

A `ConsumerStatefulWidget` that loads and saves notification preferences.

```dart
class NotificationSettingsScreen extends ConsumerStatefulWidget {
  const NotificationSettingsScreen({super.key});
  @override
  ConsumerState<NotificationSettingsScreen> createState() => _NotificationSettingsScreenState();
}
```

State management approach:
- Load preferences via `ref.watch(notificationPreferencesProvider(userId))` where userId comes from `Supabase.instance.client.auth.currentUser!.id`
- Keep a local `NotificationPreferences? _prefs` state for edits
- On any change, update local state with `copyWith` and debounce save to repository (or save on dispose/back)
- Show loading indicator while FutureProvider is loading
- Show error state with retry button if loading fails
