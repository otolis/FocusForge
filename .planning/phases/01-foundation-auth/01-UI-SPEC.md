# Phase 1 — UI Design Contract

> Visual and interaction contract for Foundation & Auth. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | Material 3 (Flutter built-in) |
| Preset | `ColorScheme.fromSeed(seedColor: Color(0xFFFF8F00))` |
| Component library | Material 3 widgets (Flutter SDK) |
| Icon library | Material Icons (built-in) — rounded variant preferred for warmth |
| Font | Nunito (headings) + Inter (body) via `google_fonts` 8.0.2 |

**Source:** CONTEXT.md locked decisions + Claude's discretion for font pairing and icon variant.

**Font rationale:** Nunito provides rounded, friendly letterforms that match the "warm & approachable" direction. Inter is highly legible at all sizes and pairs well as a neutral body font. Both are available on Google Fonts with full weight ranges.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, inline padding |
| sm | 8px | Compact element spacing, chip padding |
| md | 16px | Default element spacing, card content padding |
| lg | 24px | Section padding, form field vertical spacing |
| xl | 32px | Screen horizontal padding, section gaps |
| 2xl | 48px | Major section breaks, onboarding page spacing |
| 3xl | 64px | Top-of-screen safe area offset, hero spacing |

Exceptions:
- Touch targets: 48px minimum height (Material 3 standard, 48dp)
- Bottom navigation bar height: 80px (Material 3 default with labels)
- Onboarding illustration area: 240px fixed height

## Typography

All sizes use Flutter logical pixels (dp). Font families loaded via `google_fonts`.

| Role | Font | Size | Weight | Line Height | Usage |
|------|------|------|--------|-------------|-------|
| Body | Inter | 16dp | 400 (regular) | 1.5 | Default text, descriptions, form inputs |
| Label | Inter | 14dp | 500 (medium) | 1.4 | Button labels, captions, helper text, nav labels |
| Heading | Nunito | 22dp | 700 (bold) | 1.3 | Screen titles, section headers |
| Display | Nunito | 28dp | 800 (extrabold) | 1.2 | Onboarding titles, hero text, welcome screen |

**Implementation note:** Apply via `GoogleFonts.nunitoTextTheme()` merged with `GoogleFonts.interTextTheme()` for body styles. Override specific `TextTheme` roles:
- `displayLarge`, `displayMedium` -> Nunito 28dp/800
- `headlineMedium`, `headlineSmall` -> Nunito 22dp/700
- `bodyLarge`, `bodyMedium` -> Inter 16dp/400
- `labelLarge`, `labelMedium` -> Inter 14dp/500

## Color

### Light Mode

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFBF5` (cream white) | Scaffold background, surfaces |
| Secondary (30%) | `#FFF3E0` (warm peach tint) | Cards, input fields, bottom nav background, onboarding pages |
| Primary | Generated from seed `#FF8F00` | Primary buttons, active nav icons, selected states |
| Tertiary accent (10%) | `#2E7D6F` (teal/forest green) | Success states, streak indicators, habit check marks, onboarding illustrations accent |
| Destructive | `#D32F2F` (Material red 700) | Sign out button, delete account, error states |

### Dark Mode

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#1E1B16` (warm charcoal) | Scaffold background, surfaces |
| Secondary (30%) | `#2D2921` (warm dark surface) | Cards, input fields, bottom nav background |
| Primary | Generated from seed `#FF8F00` (soft gold in dark) | Primary buttons, active nav icons, selected states |
| Tertiary accent (10%) | `#81C784` (sage green) | Success states, streak indicators, habit check marks |
| Destructive | `#EF5350` (Material red 400) | Sign out button, delete account, error states |

**Accent reserved for (both modes):**
- Onboarding progress indicator active dot
- "Get Started" CTA button on final onboarding screen
- Success feedback (profile saved confirmation)
- Energy pattern peak-hour highlight in picker

**Implementation:** Use `ColorScheme.fromSeed` with overrides:
```
lightColorScheme = ColorScheme.fromSeed(
  seedColor: Color(0xFFFF8F00),
  brightness: Brightness.light,
  tertiary: Color(0xFF2E7D6F),
  surface: Color(0xFFFFFBF5),
)
darkColorScheme = ColorScheme.fromSeed(
  seedColor: Color(0xFFFF8F00),
  brightness: Brightness.dark,
  tertiary: Color(0xFF81C784),
  surface: Color(0xFF1E1B16),
)
```

**Source:** CONTEXT.md locked decisions for seed color, accent colors, and surface descriptions. Specific hex values derived from Material 3 tonal palette generation with manual overrides for cream/charcoal surfaces.

## Elevation & Shadow

| Element | Elevation | Shadow Style |
|---------|-----------|-------------|
| Cards | 0dp | No shadow — use surface color contrast instead (Material 3 tonal elevation) |
| Bottom navigation | 0dp | Border-top with `colorScheme.outlineVariant` at 0.5 opacity |
| Floating action button | 6dp | Material 3 default shadow |
| Bottom sheet | 1dp | Subtle lift from surface |
| Dialogs | 3dp | Standard Material 3 dialog elevation |

**Rationale:** Material 3 prefers tonal surface elevation (color shifts) over drop shadows. This creates a cleaner, warmer aesthetic matching the project vibe.

## Interaction Patterns

### Loading States

| Context | Pattern |
|---------|---------|
| Auth form submission | Disable button, replace label with CircularProgressIndicator (20dp, white on primary) |
| Google sign-in | Disable button, show loading indicator inline |
| Profile save | Show SnackBar with CircularProgressIndicator then success message |
| Screen transitions | No loading — instant navigation via go_router |
| Onboarding page swipe | Smooth physics-based PageView scroll |

### Error States

| Context | Pattern |
|---------|---------|
| Form validation | Inline error text below field, `colorScheme.error` color, Inter 12dp |
| Auth API error | SnackBar at bottom, 4-second auto-dismiss, error background with white text |
| Network error | SnackBar with retry action button |
| Profile load failure | Full-screen centered error with retry button |

### Success States

| Context | Pattern |
|---------|---------|
| Sign in / Sign up | Navigate to home (or onboarding for new users) — no explicit success message |
| Profile update | SnackBar: "Profile updated" with check icon, 2-second auto-dismiss |
| Password reset sent | Navigate to login with SnackBar: "Check your email for a password reset link." |
| Onboarding complete | Navigate to home — no explicit success message |

### Destructive Actions

| Action | Confirmation Approach |
|--------|----------------------|
| Sign out | AlertDialog with title, body, Cancel + Sign Out buttons. Sign Out button uses `colorScheme.error`. |
| Delete account | NOT in Phase 1 scope (deferred) |

### Transitions & Animation

| Transition | Type | Duration |
|------------|------|----------|
| Screen push (login -> register) | Material 3 `FadeThrough` | 300ms |
| Tab switch (bottom nav) | No animation (instant swap) | 0ms |
| Onboarding page swipe | `PageView` physics-based | Spring physics |
| Theme toggle | Implicit animation via `AnimatedTheme` | 200ms |
| Button press | `InkWell` ripple | Material default |
| SnackBar appear | Slide up from bottom | 250ms (Material default) |

## Avatar Handling

**Primary:** Initials-based avatar derived from display name.
- Background: `colorScheme.primaryContainer`
- Text: `colorScheme.onPrimaryContainer`, Nunito 20dp bold
- Fallback (no name): single letter "?" or generic person icon

**Secondary:** Optional photo upload via `image_picker`.
- Tap edit button on avatar to open image picker
- Upload to Supabase Storage `avatars` bucket
- Display uploaded photo in `CircleAvatar` with `NetworkImage`
- Show initials as fallback while photo loads

**Source:** RESEARCH.md recommendation (support BOTH initials + optional upload).

## Energy Pattern Picker Design

**Layout:** Two rows of horizontally scrollable `FilterChip` widgets.

**Peak hours row:**
- Label: "Peak Energy Hours" (Inter 14dp medium)
- Chips for hours 6 AM through 10 PM (displayed as "6 AM", "7 AM", etc.)
- Selected state: `colorScheme.primary` background, white text
- Unselected state: outlined, `colorScheme.outline` border

**Low hours row:**
- Label: "Low Energy Hours" (Inter 14dp medium)
- Same chip layout
- Selected state: `colorScheme.tertiary` background (teal/sage)
- Unselected state: same as peak unselected

**Validation:** Peak and low hours cannot overlap. If user selects a peak hour that is already low (or vice versa), auto-deselect it from the other category.

**Default:** Peak: 9 AM, 10 AM, 11 AM. Low: 2 PM, 3 PM. (from RESEARCH.md schema default)
