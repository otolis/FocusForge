# Phase 9 — UI Design Contract

> Visual and interaction contract for the Monday.com-style board table view redesign. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Flutter — not a web project) |
| Preset | not applicable |
| Component library | Material 3 (Flutter SDK) |
| Icon library | Material Icons (flutter/material.dart) |
| Font (headings) | Nunito via google_fonts — weight 700 (scoped to app-level headings outside this phase; not counted in this phase's weight budget) |
| Font (body/labels) | Inter via google_fonts — weight 400, 600 |

**Source:** Existing `lib/core/theme/text_theme.dart` and `app_theme.dart`. Nunito 700 is the app-wide heading font declared in earlier phases. This phase's table UI uses only Inter at weights 400 and 600.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps, cell internal inline padding |
| sm | 8px | Cell internal padding, row item spacing, group header icon-to-text gap |
| lg | 16px | Group header horizontal padding, card margin, section padding, input content padding, view switcher horizontal padding |
| xl | 24px | Board screen edge padding, space between view switcher and table |
| 2xl | 32px | Not used in this phase |
| 3xl | 48px | Not used in this phase |

Exceptions:
- Table row height: 36px (matches Monday.com compact row density, not a multiple of 8 but a cell rendering constant)
- Group header height: 40px (established in RESEARCH.md code examples)
- Column resize handle width: 6px (gesture target, not a spacing token)
- Sticky first column width: 200px (fixed layout constant)
- Minimum column width: 60px (resize clamp)
- Maximum column width: 400px (resize clamp)

## Color

### Surface Colors (60% Dominant + 30% Secondary)

| Role | Light Mode | Dark Mode | Usage | Split |
|------|-----------|-----------|-------|-------|
| Dominant surface (60%) | #FFFBF5 (cream) | #1E1B16 (charcoal) | Table background, page background | 60% dominant |
| Column header surface | colorScheme.surfaceContainerHighest | colorScheme.surfaceContainerHighest | Column header bar background | 30% secondary |
| Row surface (even) | #FFFBF5 | #1E1B16 | Even row background (same as dominant surface) | 30% secondary |
| Row surface (odd/zebra) | #F5F0E8 | #252118 | Odd row background for zebra striping | 30% secondary |
| Group header bars | User-assigned group color (default: #2196F3) | Same | Full-width colored header bars per group | 30% secondary |
| Row border | colorScheme.outlineVariant at 20% opacity | colorScheme.outlineVariant at 20% opacity | Horizontal row separator lines | — |

**30% secondary layer:** Group header colored bars, column header surface, and row zebra striping collectively form the mid-tone layer that provides visual structure between the dominant surface and the 10% accent highlights.

**Source:** Existing `color_schemes.dart` — cream surface #FFFBF5, charcoal #1E1B16.

### Accent and Semantic Colors (10% Accent)

| Role | Value | Usage |

| Tertiary accent | #2E7D6F (light) / #81C784 (dark) | Presence online dot (existing pattern) |

**Accent reserved for:** Active view switcher tab underline, cell edit focus border (2px), selected/hovered row highlight, primary CTA buttons only.

## Focal Point

**Primary visual anchor:** The colored `GroupHeaderWidget` bars are the focal point of the board table screen. Their user-assigned vibrant colors against the neutral cream/charcoal surface create the primary visual hierarchy, drawing the eye to group boundaries and making logical sections immediately scannable. All other elements (cells, footers, headers) are visually subordinate to these colored bars.

## Interaction Contracts

### Cell Editing State Machine

Only ONE cell can be in edit mode at a time. The table provider holds an `editingCellId` (String?) tracking which cell is active.

| State | Trigger | Visual | Behavior |
|-------|---------|--------|----------|
| Display | Default | Cell shows formatted value | Tap transitions to Editing |
| Editing | Single tap on cell | Cell shows edit control (TextField/Dropdown/Picker) with primary color focus border (2px) | Previous editing cell saves and closes first |
| Saving | Blur or Enter | Cell returns to Display with new value | Optimistic update to provider; rollback on server error |

### Column Resize

| Gesture | Behavior |
|---------|----------|
| Horizontal drag on column header right edge (6px handle) | Column width updates in real-time. Clamped to 60-400px. |
| Drag end | Column widths saved to board metadata JSONB via debounced write (500ms). |
| During resize | Horizontal scroll disabled (NeverScrollableScrollPhysics toggled on parent). |

### Column Reorder

| Gesture | Behavior |
|---------|----------|
| Long-press on column header | Header enters drag mode with slight elevation shadow. |
| Drag horizontally | Other headers shift to make room. Drop zone indicators shown. |
| Drop | Column order updated in metadata. All rows re-render column order. |

### Row Drag (Reorder within group)

| Gesture | Behavior |
|---------|----------|
| Long-press on row (500ms delay) | Row lifts with elevation, slight scale (1.02x). Matches existing Kanban drag delay. |
| Drag vertically | Other rows shift. Row constrained within its group. |
| Drop | Position updated via gap-based ordering (1000, 2000, 3000 pattern). Optimistic update. |

### Group Collapse/Expand

| Gesture | Behavior |
|---------|----------|
| Tap group header | Toggle collapsed state. Collapsed: only header visible (rows + footer hidden). Expanded: all rows + footer visible. |
| Animation | Instant visibility toggle (no height animation) to avoid jank with many rows. Chevron icon rotates 90 degrees over 150ms. |

### View Switching

| Gesture | Behavior |
|---------|----------|
| Tap "Table" tab | Switch to table view. State preserved. |
| Tap "Kanban" tab | Switch to Kanban view. Existing AppFlowyBoard renders. State preserved. |
| Animation | No transition animation — instant swap. Avoids complexity with two scroll controllers. |

### Scroll Behavior

| Region | Scroll Direction | Behavior |
|--------|------------------|----------|
| Sticky name column | Vertical only | Synced with data columns vertical scroll via linked ScrollControllers. |
| Data columns | Vertical + Horizontal | Vertical synced with name column. Horizontal scroll independent. |
| Group headers | Horizontal | Span full width (sticky + scrollable). Scroll with horizontal position. |

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Minimum touch target | 36px height for table rows (meets content density exception). All standalone buttons/icons: 48px. |
| Color contrast (pills) | White text (#FFFFFF) on all pill colors must meet WCAG AA (4.5:1). Monday red #E2445C passes (4.6:1). Monday orange #FDAB3D: use dark text (#1E1B16) instead of white for AA compliance. Monday blue #579BFC: use dark text for AA compliance. Monday gray #C4C4C4: use dark text (#1E1B16) for AA compliance. |
| Screen reader: cells | Each cell announces its column name + value (e.g., "Status: Working on it"). Use Semantics widget with label. |
| Screen reader: groups | Group header announces "Group: {name}, {count} items, {expanded/collapsed}". |
| Screen reader: row drag | Row announces "Item {title}, position {n} of {total} in group {name}". |
| Reduce motion | Chevron rotation respects `MediaQuery.disableAnimations`. Instant toggle if reduce motion enabled. |

**Contrast exception for priority/status pills:** If the computed contrast ratio of white text on a pill color is below 4.5:1, switch text to dark (#1E1B16 in light mode, #FFFBF5 in dark mode). Implement via a `contrastTextColor(Color background)` utility that returns white or dark based on relative luminance.
