`class TimelineConstants` with static constants and methods:

- `static int yToSnappedMinute(double y)` -- converts Y offset to nearest 15-min mark: `final rawMinutes = (y / pixelsPerMinute).round() + startHour * 60;` then `(rawMinutes / snapMinutes).round() * snapMinutes;` Clamp result between `startHour * 60` and `endHour * 60`.
- `static EnergyZone getZone(int hour, EnergyPattern pattern)` -- returns `peak` if `pattern.peakHours.contains(hour)`, `low` if `pattern.lowHours.contains(hour)`, else `regular`.
- `static List<ScheduleBlock> resolveOverlaps(List<ScheduleBlock> blocks)` -- sorts by startMinute, walks sequentially: if block overlaps previous (block.startMinute < prev.endMinute), push block.startMinute to prev.endMinute. Clamp endMinute to `endHour * 60`. Returns new list with adjusted blocks.

**2. `supabase/functions/generate-schedule/index.ts`** -- Edge Function:

Uses `Deno.serve(async (req: Request) => { ... })` pattern.

Handle CORS preflight: if `req.method === 'OPTIONS'`, return `new Response('ok', { headers: corsHeaders })`.

In try block:
- Parse request body: `const { items, energyPattern, constraints } = await req.json()`
- Get `GROQ_API_KEY` from `Deno.env.get('GROQ_API_KEY')`, throw if missing
- Build system prompt via `buildSystemPrompt(energyPattern, constraints)`:
  - Convert peak_hours and low_hours arrays to human-readable time strings (e.g., "9AM, 10AM, 11AM")
  - Include rules: schedule between 6 AM (360 min) and 10 PM (1320 min), HIGH energy items in peak hours, LOW energy items in low hours, MEDIUM anywhere, leave 15-30 min gaps, no overlaps, all times in minutes from midnight
  - Include: "Respond with valid JSON" instruction for json_object mode
  - Format: `{ "blocks": [{ "item_id": "...", "title": "...", "start_minute": N, "duration_minutes": N, "energy_level": "high|medium|low" }] }`
  - If constraints provided, append as additional instruction
- Build user prompt via `buildUserPrompt(items)`:
  - Format each item as: `- ID: {id}, Title: "{title}", Duration: {duration_minutes} min, Energy: {energy_level}`
  - Wrap with: `Schedule these items for today:\n{itemList}`
- Call Groq API via fetch to `https://api.groq.com/openai/v1/chat/completions`:
  - Headers: `Authorization: Bearer {key}`, `Content-Type: application/json`
  - Body: model `llama-3.3-70b-versatile`, messages (system + user), temperature 0.3, max_completion_tokens 2048, response_format `{ type: 'json_object' }`
- If response not ok, throw with status and error text
- Parse `completion.choices[0].message.content` as JSON
- Return parsed schedule JSON with 200 status and corsHeaders + Content-Type

In catch block: return error JSON `{ error: error.message }` with 500 status and corsHeaders.
